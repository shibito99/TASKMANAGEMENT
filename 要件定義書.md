## 0. エグゼクティブサマリー

### 背景と目的

株式会社〇〇（以下、お客さま）では、社内のタスク管理をチャットや口頭で行っており、
「誰が何をどこまで進めているか」が把握しにくい状況でした。

本プロジェクトでは、タスクをボード・リスト・カードで視覚的に管理できるWebアプリを開発・納品することで、
チーム全体の作業状況を一目で把握できる環境を提供します。

### ターゲットユーザー

| 項目 | 内容 |
|---|---|
| 想定利用者 | 小規模チーム（5名以下）のプロジェクトメンバー |
| IT リテラシー | 普通（スマートフォンやExcelは日常的に使用） |
| 利用シーン | 日々の業務タスクの進捗確認・管理 |

### 解決する課題

- **課題①：** タスクの抜け漏れが発生しやすい
  → カードで一元管理することで、やるべきことを可視化する
- **課題②：** 進捗状況が把握できない
  → リスト（To Do / In Progress / Done）でステータスを一目で確認できる
- **課題③：** どのツールを使えばよいか分からない
  → ブラウザだけで動作するシンプルなアプリを提供する

## 1. プロジェクト概要

- **プロジェクト名：** タスク管理アプリ
- **目的：** ボード・リスト・カードでタスクを視覚的に管理する
- **対象ユーザー：** 小規模チームのプロジェクトメンバー
- **動作環境：** ローカル環境（ブラウザで起動）
- **技術スタック：** 詳細は[技術設計書.md](技術設計書.md)を参照

## 2. ユースケース

ユーザーがアプリで行う操作を一覧にまとめます。

| ID | ユースケース名 | 概要 |
|---|---|---|
| UC-01 | リストを追加する | 新しいリスト（列）を作成する |
| UC-02 | リストを削除する | 不要なリストを削除する |
| UC-03 | リスト名を変更する | リストの名前を編集する |
| UC-04 | カードを追加する | リストにタスクカードを追加する |
| UC-05 | カードを削除する | 不要なカードを削除する |
| UC-06 | カードを編集する | カードのタイトルを変更する |
| UC-07 | カードを移動する | カードを別のリストへドラッグ&ドロップで移動する |
| UC-08 | カード詳細を開く | カードをクリックして詳細情報を確認・編集する |
| UC-09 | 期限日を設定する | カード詳細から期限日を入力する |
| UC-10 | ラベルを付ける | カード詳細からラベル（色）を選択して付与する |

## 3. 操作フロー図

### 3-1. タスクを追加して進捗を更新するフロー（基本フロー）

```
[アプリを開く]
      │
      ▼
[ボード画面が表示される]
      │
      ├─── リストがない場合 ──→ [＋リスト追加をクリック] ──→ [リスト名を入力] ──→ [リストが作成される]
      │                                                                                    │
      │◀───────────────────────────────────────────────────────────────────────────────────┘
      │
      ▼
[リスト内の「＋カード追加」をクリック]
      │
      ▼
[カードのタイトルを入力して追加]
      │
      ▼
[カードが「To Do」リストに表示される]
      │
      ▼
[作業開始 → カードを「In Progress」へドラッグ&ドロップ]
      │
      ▼
[作業完了 → カードを「Done」へドラッグ&ドロップ]
      │
      ▼
[完了]
```

---

### 3-2. カード詳細を編集するフロー

```
[ボード画面でカードをクリック]
      │
      ▼
[カード詳細ポップアップが開く]
      │
      ├─── タイトルを変更したい ──→ [タイトル欄を編集]
      │
      ├─── 説明を書きたい ────────→ [説明欄にテキストを入力]
      │
      ├─── 期限日を設定したい ────→ [期限日欄に日付を入力]
      │
      └─── ラベルを付けたい ──────→ [ラベルの色を選択]
                │
                ▼
          [「保存」ボタンをクリック]
                │
                ▼
          [ポップアップが閉じてカードに反映される]
```

---

### 3-3. カードを削除するフロー

```
[ボード画面でカードをクリック]
      │
      ▼
[カード詳細ポップアップが開く]
      │
      ▼
[「削除」ボタンをクリック]
      │
      ▼
[確認メッセージが表示される「本当に削除しますか？」]
      │
      ├─── はい ──→ [カードが削除される] ──→ [ポップアップが閉じる]
      │
      └─── キャンセル ──→ [ポップアップに戻る]
```

## 4. 機能要件

> **凡例：** `Must` = 必須機能（必ず実装）／ `Want` = 追加機能（余裕があれば実装）

### ボード機能

- **B-01** ボードの表示 `Must`
  - リストとカードを一覧表示する
  - 初期リリースはボードは1つのみ（複数ボードは対象外）

---

### リスト機能

- **L-01** リストの追加 `Must`
  - 任意の名前でリストを新規追加できる
- **L-02** リストの削除 `Must`
  - リストを削除できる（内包するカードも同時に削除）
- **L-03** リスト名の編集 `Must`
  - リスト名をクリックして直接編集できる
- **L-04** リストの並び替え `Want`
  - ドラッグ&ドロップでリストの順序を変更できる

---

### カード機能

- **C-01** カードの追加 `Must`
  - リスト内に任意のタイトルでカードを追加できる
- **C-02** カードの削除 `Must`
  - カードを削除できる
- **C-03** カードタイトルの編集 `Must`
  - カードのタイトルを編集できる
- **C-04** カードの移動 `Must`
  - ドラッグ&ドロップでカードを別リストへ移動できる
- **C-05** カードの詳細表示 `Want`
  - カードをクリックするとポップアップ画面を表示できる
- **C-06** 説明文の追加 `Want`
  - カード詳細に説明文を記入できる
- **C-07** 期限日の設定 `Want`
  - カード詳細に期限日を設定できる
- **C-08** ラベルの付与 `Want`
  - カードに色付きラベルを付けられる

---

### データ保存機能

- **D-01** データの永続化 `Must`
  - ページを再読み込みしてもデータが消えない
  - バックエンドのSQLiteデータベースに保存する

4. カードをクリックすると詳細ポップアップを開く

## 6. 非機能要件

### 6-1. 動作環境

- **対応ブラウザ：** Google Chrome 最新版
- **対応デバイス：** PC（Windows）のみ
- **インターネット：** 不要（ローカル環境で完結）

### 6-2. パフォーマンス

- ページの初期表示が **3秒以内** に完了すること
- カードの追加・移動などの操作が **1秒以内** に反映されること

### 6-3. データ

- アプリを再起動してもデータが保持されること
- データはローカルのSQLiteファイルに保存し、外部サーバーには送信しないこと

### 6-4. ユーザビリティ

- ITリテラシーが一般的なユーザーが、説明なしで基本操作（カードの追加・移動）を行えること
- ボタンやリンクのクリック可能領域は十分な大きさを確保すること（最小 44px × 44px）
- カードの削除など、取り消しのきかない操作の前に確認メッセージを表示すること

### 6-5. 保守性

- コードは機能ごとにファイルを分割し、後から修正・追加しやすい構成にすること

## 7. スコープ外（今回は作らないもの）

- ユーザー認証（ログイン・パスワード管理）
- 複数ボードの管理
- 複数人での共同編集・リアルタイム同期
- スマートフォン・タブレット対応
- クラウド同期・外部サーバーへのデータ保存
- 通知・リマインダー機能
- ファイル添付機能
